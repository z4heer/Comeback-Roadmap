# 90-Day Career Comeback Plan — Python Backend Engineer

Goal: Become interview-ready for mid‑level Python backend roles within 90 days by focusing on core fundamentals, a flagship project, and interview preparation.

Overview
- Weeks 1–4: Core refresh (Python, SQL, systems fundamentals, testing)
- Weeks 5–8: Flagship project (design, implement, deploy)
- Weeks 9–12: Polish, interview prep, apply

Week 1 — Foundations (Python & Git)
- Daily: 1–2 hours of focused practice
- Tasks:
  - Refresh Python basics: data structures, comprehensions, OOP
  - Git workflow: branching, PRs, basic rebasing
  - Write 3 small exercises with tests

Week 2 — Web & APIs
- Tasks:
  - Build a minimal REST API with Flask or FastAPI (CRUD)
  - Add unit tests and basic request validation
  - Short doc: API design decisions

Week 3 — Databases & SQL
- Tasks:
  - Design schema for a small app (flagship domain)
  - Implement migrations (Alembic)
  - Add queries and indexing notes

Week 4 — Testing, CI, and Docker
- Tasks:
  - Add tests (unit + integration)
  - Dockerize app and add GitHub Actions CI pipeline
  - Create README for the app

Weeks 5–8 — Flagship Project
- Week 5: Architecture & core backend
- Week 6: Persistence, background jobs, caching
- Week 7: Authentication, rate limiting & security notes
- Week 8: End-to-end tests, deployment to a cloud or Docker Compose

Weeks 9–12 — Polish & Interview Prep
- Week 9: Performance tuning, profiling, caching
- Week 10: System design study & writeups
- Week 11: Behavioral + technical interview practice
- Week 12: Applications, follow-ups, and interview scheduling

Deliverables by day 90
- Working flagship backend with README and docs
- 8–12 well-scoped GitHub issues completed
- Interview Q&A docs (python-qna.md, system-design.md)
- Short portfolio README summarizing project, responsibilities, and learnings

Metrics (weekly check)
- Completed tasks vs planned
- Number of practice interview problems attempted
- Pull requests opened / merged
